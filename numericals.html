1 · Sampling, quantization & storage Session 2

◆ Understand it

Sampling chooses how many pixels (an \(M\times N\) grid → spatial resolution); quantization chooses how many gray levels each pixel can hold (\(L = 2^b\) → intensity resolution). Storage is just every pixel times its bit-depth.

Method

Gray levels
\(L = 2^{b}\) for \(b\) bits.
Storage in bits
\(\text{bits} = M \times N \times b\) (×3 for an RGB color image).
Convert
÷8 → bytes, ÷1024 → KB, ÷1024 again → MB.

Worked example — exam style

Q. A \(256\times256\) grayscale image is stored at 8 bits/pixel. Number of gray levels? Uncompressed size?

Levels
\(L = 2^{8} = 256\) gray levels (0–255).
Bits
\(256\times256\times8 = 524{,}288\) bits.
Bytes / KB
\(\div 8 = 65{,}536\) bytes \(= 64\) KB.
Answer

256 levels; 65 536 bytes = 64 KB. (24-bit color would be ×3 = 192 KB.)

✦ Tip

"How many bits for \(N\) levels?" → \(b=\log_2 N\). Watch units: examiners love asking for KB vs MB. Too few samples ⇒ pixelation; too few levels ⇒ false contouring.

2 · Contrast stretching Session 2

◆ Understand it

An image using only part of the scale looks flat. A linear stretch rescales the used range \([r_{\min}, r_{\max}]\) onto the whole range \([0, L-1]\) — a straight line on the \(s\)-vs-\(r\) graph.

Method

Write the formula
\(s = (r - r_{\min})\cdot\dfrac{L-1}{r_{\max}-r_{\min}}\)
Compute the slope
\(\dfrac{L-1}{r_{\max}-r_{\min}}\) once.
Substitute & round
plug the pixel value, round to nearest integer.

Worked example — exam style

Q. Pixels span [50, 150]. Stretch to [0, 255]. Find \(s\) for \(r=100\).

Slope
\(\dfrac{255}{150-50} = 2.55\)
Substitute
\(s = (100-50)\times 2.55 = 127.5\)
Answer

\(s = 127.5 \approx \mathbf{128}\).

✦ Tip

To invert (given output, find input): \(r = \frac{s}{\text{slope}} + r_{\min}\). Check endpoints: \(r_{\min}\!\to\!0\) and \(r_{\max}\!\to\!255\) must hold.

3 · Log transform Session 2/3

◆ Understand it

\(s = c\,\log(1+r)\) expands dark values and compresses bright ones — useful for huge-dynamic-range data (e.g. Fourier spectra). The \(+1\) avoids \(\log 0\). Choose \(c\) so the largest input maps to the largest output.

Method

Pick c so max→max
\(c = \dfrac{L-1}{\log(1 + (L-1))}\)
Apply
\(s = c\,\log(1+r)\), then round.

Worked example — exam style

Q. 8-bit image. Choose \(c\) so \(r=255\) maps to 255, then find \(s\) for \(r=100\) (natural log).

Find c
\(c = \dfrac{255}{\ln(256)} = \dfrac{255}{5.545} = 45.99\)
Apply to r = 100
\(s = 45.99\times \ln(101) = 45.99\times 4.615 = 212.3\)
Answer

\(c \approx 45.99\); \(\;s \approx \mathbf{212}\). Note how a mid input (100) is pushed up to 212 — dark detail brightened.

✦ Tip

State your log base. With base-10, \(c = 255/\log_{10}(256) = 105.9\). Either is accepted if consistent; verify \(r=255\Rightarrow s=255\).

4 · Power-law (gamma) Session 2/3

◆ Understand it

\(s = c\,r^{\gamma}\). On normalised values, \(\gamma<1\) brightens (expands dark detail), \(\gamma>1\) darkens (expands bright detail), \(\gamma=1\) is identity. This is exactly gamma correction for displays.

Method

Normalise
\(r_n = r/(L-1)\) so it lies in [0,1].
Apply power
\(s_n = r_n^{\gamma}\) (with \(c=1\)).
Scale back
\(s = (L-1)\,s_n\), round.

Worked example — exam style

Q. 8-bit image, \(\gamma=0.5\), \(c=1\). Find \(s\) for \(r=100\).

Normalise
\(r_n = 100/255 = 0.392\)
Power
\(s_n = 0.392^{0.5} = \sqrt{0.392} = 0.626\)
Scale
\(s = 255\times 0.626 = 159.7\)
Answer

\(s \approx \mathbf{160}\) (brightened, since \(\gamma=0.5<1\)). For \(\gamma=2\): \(255\times0.392^2 = 39.2\) → darkened.

5 · Histogram equalization Session 3

◆ Understand it

Push every pixel through the scaled CDF of its own histogram: \(s_k=(L-1)\,\text{CDF}(r_k)\). Crowded intensities (steep CDF) get spread apart → contrast up, histogram flattened.

Method

Histogram
count \(n_k\) for each level; total \(MN\).
PDF then CDF
\(p_k = n_k/MN\); cumulative sum \(\to\) CDF.
Map
\(s_k = \text{round}\big((L-1)\cdot\text{CDF}_k\big)\).
Replace
swap each old level for its new \(s_k\).

Worked example — exam style (class 5×5, L=8)

Q. 25 pixels; only levels 3 (×6), 4 (×14), 5 (×5) occur. Equalize.

rnkCDF7·CDFnew s
366/25=0.241.682
41420/25=0.805.606
5525/25=1.007.007
Answer

Map \(3\to2,\ 4\to6,\ 5\to7\) — the three levels spread out across the range.

6 · Convolution / applying a kernel Session 3/4

◆ Understand it

Centre the kernel on the pixel, multiply each kernel weight by the pixel under it, and sum. (Convolution flips the kernel 180° first; correlation does not. For symmetric kernels they're identical.) The single number you get is the filtered value at that pixel.

Method

Align
place the kernel centre on the target pixel.
(Convolution only) flip
rotate the kernel 180°. Many edge masks are anti-symmetric, so be careful with signs.
Multiply & add
sum of (weight × underlying pixel) over the 9 cells.

Worked example — exam style

Q. Apply \(\begin{bmatrix}-1&0&1\\-1&0&1\\-1&0&1\end{bmatrix}\) to the centre of \(\begin{bmatrix}10&10&200\\10&10&200\\10&10&200\end{bmatrix}\).

Per row: (−1)·left + 0·mid + (1)·right
each row: \(-10 + 200 = 190\)
Sum the three rows
\(190+190+190 = 570\)
Answer

Response \(= \mathbf{570}\) — a strong vertical-edge response. (If the patch were left-right symmetric, the result would be 0 — as in midsem Q3.)

7 · Averaging (box) filter Session 3

◆ Understand it

A \(3\times3\) box filter replaces a pixel with the mean of its 9-pixel neighbourhood — simple smoothing. Kernel is \(\frac{1}{9}\) times all-ones.

Worked example — exam style

Q. Smooth the centre of \(\begin{bmatrix}10&20&30\\40&50&60\\70&80&90\end{bmatrix}\) with a \(3\times3\) average filter.

Sum all 9
\(10{+}20{+}30{+}40{+}50{+}60{+}70{+}80{+}90 = 450\)
Divide by 9
\(450/9 = 50\)
Answer

New centre value \(= \mathbf{50}\). (Here it equals the centre pixel because the patch is a symmetric ramp.)

✦ Tip

For a weighted (Gaussian) kernel, multiply each pixel by its weight, sum, then divide by the sum of weights. Box = all weights equal.

8 · Gradient: magnitude & direction Session 4

◆ Understand it

Apply a \(g_x\) mask and a \(g_y\) mask (Prewitt/Sobel). Combine into magnitude (edge strength) and direction. The gradient points across the edge; the edge runs perpendicular to it.

Method

Compute gx, gy
convolve patch with the two masks.
Magnitude
\(\|\nabla f\| = \sqrt{g_x^2 + g_y^2}\) (or \(|g_x|+|g_y|\)).
Direction
\(\theta = \tan^{-1}(g_y/g_x)\); edge direction \(=\theta\pm90^\circ\).

Worked example — exam style (Sobel)

Q. Patch \(\begin{bmatrix}100&100&100\\100&100&200\\100&200&200\end{bmatrix}\). Find \(g_x, g_y\), magnitude and direction (Sobel).

gx = (col₃·[1,2,1]) − (col₁·[1,2,1])
\((100{+}400{+}200) - (100{+}200{+}100) = 700 - 400 = 300\)
gy = (row₃·[1,2,1]) − (row₁·[1,2,1])
\((100{+}400{+}200) - (100{+}200{+}100) = 700 - 400 = 300\)
Magnitude
\(\sqrt{300^2+300^2} = 300\sqrt2 \approx 424.3\)
Direction
\(\theta = \tan^{-1}(300/300) = 45^\circ\); edge \(= 135^\circ\)
Answer

\(g_x=g_y=300\), \(\|\nabla f\|\approx424\), gradient at \(45^\circ\), so the edge runs at \(135^\circ\) (a diagonal).

9 · Laplacian (4 vs 8 neighbour) Session 4

◆ Understand it

A second-derivative, isotropic operator: \(-c\) on the centre, \(+1\) on the neighbours. It's 0 in flat/linear regions and flips sign across an edge (the edge = zero-crossing). 8-neighbour reacts more strongly (and to diagonals) than 4-neighbour.

Worked example — exam style (5×5, edge 150|50)

Q. Every row = [150,150,150,50,50]. Laplacian at a bright-side edge pixel (value 150; right neighbour = 50).

K₁ (4-neighbour): −4·centre + (up+down+left+right)
\(-4(150) + (150{+}150{+}150{+}50) = -600+500 = -100\)
K₂ (8-neighbour): −8·centre + (8 neighbours: 5×150 + 3×50)
\(-8(150) + 900 = -1200+900 = -300\)
Answer

\(K_1=-100,\ K_2=-300\). On the dark side these become \(+100\) and \(+300\) — the sign flip locates the edge (zero-crossing). \(K_2\) is ~3× stronger (more sensitive, also noisier).

10 · Hough transform Session 5

◆ Understand it

Each image point \((x,y)\) maps to a vote-line \(c=-x\,m+y\) in \((m,c)\) space. Collinear points ⇒ their vote-lines meet at one \((m,c)\) = the shared line. For vertical lines (slope ∞) switch to polar \(\rho=x\cos\theta+y\sin\theta\) (sinusoids that intersect).

Method (collinearity / line params)

Write each point's vote-line
\(c = -x_i\,m + y_i\).
Intersect any two
solve for \((m,c)\).
Verify with the rest
substitute; if all pass → collinear, line \(y=mx+c\).
If vertical (parallel vote-lines)
use \(\rho\text{–}\theta\): set sinusoids equal, solve \(\theta\), back-substitute \(\rho\).

Worked example — exam style

Q. Are \((1,2),(2,3),(3,4)\) collinear (Hough)?

Vote-lines
\(c=-m+2,\ \ c=-2m+3,\ \ c=-3m+4\)
Intersect first two
\(-m+2=-2m+3 \Rightarrow m=1,\ c=1\)
Check third
\(-3(1)+4 = 1\) ✓
Answer

Collinear; common line \(y = x + 1\) at \((m,c)=(1,1)\).

11 · Harris corner response Session 6

◆ Understand it

Build \(M=\sum\begin{bmatrix}I_x^2 & I_xI_y\\ I_xI_y & I_y^2\end{bmatrix}\) over a window, then \(R=\det M - k(\text{trace }M)^2\). \(R>0\) corner, \(R<0\) edge, \(|R|\) small flat (\(k=0.04\text{–}0.06\)).

Method

Assemble M
from \(\sum I_x^2,\ \sum I_y^2,\ \sum I_xI_y\).
det & trace
\(\det = (\sum I_x^2)(\sum I_y^2) - (\sum I_xI_y)^2\); \(\text{trace}=\sum I_x^2 + \sum I_y^2\).
Response
\(R = \det - k\,(\text{trace})^2\); classify by sign.

Worked example — exam style

Q. A window gives \(\sum I_x^2=2,\ \sum I_y^2=2,\ \sum I_xI_y=0\). Take \(k=0.04\). Classify.

M, det, trace
\(M=\begin{bmatrix}2&0\\0&2\end{bmatrix}\), \(\det=4\), \(\text{trace}=4\)
R
\(R = 4 - 0.04(4)^2 = 4 - 0.64 = 3.36\)
Answer

\(R = 3.36 > 0\) → corner (both eigenvalues \(=2\), large). An edge case \(\sum I_x^2=2,\sum I_y^2=0\) gives \(R=-0.16<0\).

12 · HOG histogram & binning Session 6

◆ Understand it

For each pixel compute gradient magnitude and orientation, then drop its magnitude into a 9-bin histogram over 0–180°. The standard scheme splits each vote between the two nearest bin centres (interpolation).

Method (interpolated binning)

Bin centres
0,20,40,…,160 (width 20°).
Locate θ
find the two centres it falls between.
Weights
weight to a centre \(= \dfrac{\text{distance to the }far\text{ centre}}{20}\).
Distribute
add weight × |g| to each of the two bins.

Worked example — exam style

Q. A pixel has direction 57° and magnitude 85. Bin it.

Between which centres?
57° lies between 40° and 60°.
Weights
to 40°: \((60-57)/20=0.15\); to 60°: \((57-40)/20=0.85\)
Distribute |g| = 85
bin 40° \(+= 0.15\times85 = 12.75\); bin 60° \(+= 0.85\times85 = 72.25\)
Answer

Bin 40° = 12.75, Bin 60° = 72.25, rest 0. (Class "simple" binning instead dumps all 85 into the single 40–60° bin.)

13 · RGB → CMYK conversion Session 3

◆ Understand it

Print is subtractive: \(C=1-R,\ M=1-G,\ Y=1-B\) (on normalised 0–1 values). Then pull out black \(K=\min(C,M,Y)\) to save colored ink.

Worked example — exam style

Q. Convert \((R,G,B)=(0.2,0.5,0.9)\) to CMYK (course convention: subtract \(K\)).

CMY
\(C=0.8,\ M=0.5,\ Y=0.1\)
K
\(K=\min(0.8,0.5,0.1)=0.1\)
Subtract K
\(C=0.7,\ M=0.4,\ Y=0.0\)
Answer

\((C,M,Y,K)=(0.7,\,0.4,\,0.0,\,0.1)\). (Normalised variant divides by \(1-K\): \(C'=0.7/0.9=0.78\), etc.)

14 · RANSAC iteration count Session 7

◆ Understand it

How many random samples \(N\) guarantee (with probability \(p\)) at least one all-inlier sample, given an outlier fraction \(e\) and minimal sample size \(s\)?

Method

Prob. one sample is all-inliers
\((1-e)^{s}\).
Formula
\(N = \dfrac{\log(1-p)}{\log\!\big(1-(1-e)^{s}\big)}\), round up.

Worked example — exam style

Q. Line fitting (\(s=2\)), outliers \(e=0.2\), want \(p=0.99\). Find \(N\).

All-inlier prob.
\((1-0.2)^2 = 0.64\)
Plug in
\(N = \dfrac{\log(0.01)}{\log(1-0.64)} = \dfrac{-2}{\log(0.36)} = \dfrac{-2}{-0.4437} = 4.5\)
Answer

\(N \approx \mathbf{5}\) iterations. More outliers or larger \(s\) ⇒ steeply more iterations.

15 · Canny edge detector — all 4 steps Session 5

◆ Understand it

Canny isn't one formula — it's a 4-stage pipeline run on the whole image. A full worked numerical (Gaussian smoothing → Sobel magnitude/angle → non-maximum suppression → double-threshold hysteresis) on a \(5\times5\) image with a noisy pixel is solved end-to-end, with every intermediate matrix, on the Session 5 page.

Method (per stage)

① Smooth
convolve with a \(3\times3\) Gaussian \(\frac1{16}[1\,2\,1;2\,4\,2;1\,2\,1]\) → noise (e.g. a 160 speck) collapses toward its neighbours.
② Gradient
Sobel \(g_x,g_y\) → magnitude \(\sqrt{g_x^2+g_y^2}\) and angle \(\tan^{-1}(g_y/g_x)\), quantised to \(\{0,45,90,135\}^\circ\).
③ NMS
keep a pixel only if its magnitude ≥ its two neighbours along the gradient direction → 1-px-thin edges.
④ Hysteresis
strong if \(\ge T_H\), weak if \(\in[T_L,T_H)\); keep weak only if 8-connected to a strong edge (ratio \(T_H:T_L\approx\)2:1–3:1).
