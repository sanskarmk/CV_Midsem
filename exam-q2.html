Question 2 · 5 Marks

Intensity Transforms, Hough Mapping & Gaussian

📄 Original question

Q2(a) An image suffers from low contrast and has some noise. Explain how you would combine two different intensity transformation techniques to improve the image quality. Justify your choice of methods. 2M

Q2(b) A point \((x_0,y_0)=(3,4)\) in the image space maps to a line in Hough space using \(c = -x_0 m + y_0\). Calculate \(c\) for \(m=2\) and \(m=-1\). 2M

Q2(c) Why is a Gaussian filter often preferred over a simple average filter? 1M

Solution — Q2(a)

Two problems, so combine two operations and — crucially — order them correctly:

First: reduce noise (smoothing)
Apply a Gaussian smoothing filter to suppress the noise. This must come first, because any contrast-boosting step amplifies whatever noise is present.
Then: boost contrast
Apply histogram equalization (or piecewise contrast stretching) to spread the now-clean intensities across the full range, restoring visible detail.
Model answer

Combine smoothing then contrast enhancement: Gaussian smoothing first to remove noise, then histogram equalization / contrast stretching to fix the low contrast. Justification: contrast enhancement multiplies differences between neighbouring values, so if applied before denoising it would magnify the noise too. Denoising first, enhancing second, gives a clean and high-contrast result. (If the noise is salt-and-pepper, prefer a median filter for the first step.)

Solution — Q2(b)

The Hough mapping sends image point \((x_0,y_0)\) to the parameter-space line \(c = -x_0\,m + y_0\). With \((x_0,y_0)=(3,4)\): \(c = -3m + 4\).

m = 2
\(c = -3(2) + 4 = -6 + 4 = \mathbf{-2}\)
m = −1
\(c = -3(-1) + 4 = 3 + 4 = \mathbf{7}\)
Answer

\(c=-2\) when \(m=2\); \(\;c=7\) when \(m=-1\). These two \((m,c)\) points both lie on the single vote-line contributed by image point \((3,4)\).

Solution — Q2(c)

Answer

A Gaussian filter weights pixels by distance from the centre, so it produces a smoother, artefact-free result (no blocky edges or ringing like the box filter). It is isotropic (blurs equally in all directions), separable (an \(n\times n\) Gaussian = two 1-D passes, so \(2n\) vs \(n^2\) multiplies), and has a clean low-pass frequency response (its Fourier transform is another Gaussian, with no negative side-lobes). It therefore suppresses noise while preserving structure better than a simple average.

\(c=-2m+3\).

P2.2 Point \((5,2)\) maps to Hough space. Compute \(c\) at \(m=0\) and \(m=3\), and state the geometric meaning of the \(m=0\) value. 2M

\(c=-5m+2\).

m = 0
\(c=2\) — the intercept of a horizontal line \(y=2\) through the point.
m = 3
\(c=-15+2 = \mathbf{-13}\)
Answer

\(c=2\) at \(m=0\) (a horizontal line through \((5,2)\) has intercept 2); \(c=-13\) at \(m=3\).

P2.3 An image has salt-and-pepper noise and is too dark. Which two operations, in what order, and why? 2M
Answer

(1) Median filter first — it removes impulse (salt-and-pepper) noise far better than a linear filter because the outlier pixel is replaced by a neighbourhood median, not averaged in. (2) Gamma correction with \(\gamma<1\) (or log transform / equalization) to brighten the dark image. Denoise first so brightening doesn't amplify the impulses.

P2.4 Give one situation where a box (average) filter is actually preferable to a Gaussian. 1M
Answer

When speed/simplicity matters most: a box filter can be implemented with an integral image (summed-area table) so its cost is independent of kernel size — ideal for very large kernels or real-time use where its mild artefacts are acceptable. (It's also exactly what's needed if you genuinely want a uniform local average.)

P2.5 Why does applying contrast stretching before denoising give a worse result than the reverse order? 1M
Answer

Contrast stretching multiplies the spread between intensities. Noise is a small intensity deviation, so stretching first amplifies the noise along with the signal, making it harder to remove afterwards. Denoising first keeps the subsequent stretch operating on a clean signal.
