Canny edge detection — the 4 steps

Canny is "our first full computer-vision pipeline" and still a workhorse. It runs four stages:

1 · Smooth with a Gaussian
Blur the image with a Gaussian to suppress noise before differentiating (differentiation amplifies noise). In practice we combine "blur then differentiate" into a single derivative-of-Gaussian filter using the associativity of convolution — saving an operation.
2 · Gradient magnitude & angle
Apply Sobel to get \(g_x,g_y\); compute \(\|\nabla f\|=\sqrt{g_x^2+g_y^2}\) and \(\theta=\tan^{-1}(g_y/g_x)\) at every pixel. Edges are now thick ridges.
3 · Non-maximum suppression (NMS)
Thin the ridges to 1-pixel width: keep a pixel only if its magnitude is a local maximum along the gradient direction (compare against the two neighbours in that direction). The angle is snapped to one of 4 directions: 0°, 45°, 90°, 135°.
4 · Double thresholding + hysteresis
Use two thresholds \(T_L, T_H\). Pixels \(\ge T_H\) are strong edges (definitely edges); pixels between \(T_L\) and \(T_H\) are weak — kept only if connected (via the 8-neighbourhood) to a strong edge. This links broken edges while rejecting noise. Recommended ratio \(T_H:T_L \approx\) 2:1 to 3:1.
◆ Why two thresholds?

A single threshold either lets noise through (too low) or breaks real edges into dashes (too high). Hysteresis gets the best of both: a high threshold to confidently start an edge, and a low threshold to continue it as long as it stays connected. "Strong edges are edges; weak edges are edges iff they touch a strong one."

Step 3 — Non-maximum suppression

Keep a pixel only if its magnitude is ≥ its two neighbours along the gradient direction. The edge pixels have direction \(0^\circ\) → compare left/right.

Why line fitting needs voting

After edge detection we have lots of edge points, and we want the lines behind them. Why not just fit? Because of three problems: clutter (extra points, multiple lines — which points belong to which line?), missing evidence (only parts of each line survive — how to bridge gaps?), and noise (points and orientations are imperfect). Checking every subset of points is infeasible.

◆ The voting idea

Let every feature vote for all models compatible with it; then look for models that collect many votes. Noise and clutter also vote, but their votes are scattered and inconsistent, so they don't accumulate. The Hough transform is exactly this idea for lines.

Hough transform in (m, c) space

A line is \(y = mx + c\). The Hough trick is a duality between image space and parameter space:

Image space \((x,y)\)Parameter space \((m,c)\)
A line \(y=mx+c\)A single point \((m,c)\)
A point \((x_0,y_0)\)A line \(c = -x_0\,m + y_0\)
Points on one common lineLines through one common point

So: each edge point \((x_0,y_0)\) "votes" by drawing the line \(c=-x_0 m+y_0\) in parameter space. Where many of these vote-lines intersect, that \((m,c)\) is the line shared by many points. We accumulate votes in a discrete grid of \((m,c)\) bins and pick the peaks.

The polar (ρ, θ) representation

The \((m,c)\) form has a fatal flaw: vertical lines have infinite slope (\(m\to\infty\)), which can't fit in a finite accumulator. The fix is the polar/normal form:

\[ \rho = x\cos\theta + y\sin\theta \]
  • \(\rho\) — perpendicular distance from the line to the origin.
  • \(\theta\) — angle the perpendicular makes with the x-axis.

Now both parameters are bounded (\(\theta\in[0,180^\circ)\), \(\rho\in[-D, D]\)), so every line — vertical included — fits in the accumulator. A point \((x_0,y_0)\) in image space maps to a sinusoid in \((\rho,\theta)\) space; intersecting sinusoids vote for a common line. This is the form actually used in practice.

Worked numerical — are the points collinear?

The class example (and the template for Q5): determine whether \((1,2), (2,3), (3,4)\) lie on one line using the Hough transform.

Each point maps to a line \(c = -x\,m + y\) in \((m,c)\) space:

Write the vote-line for each point
\((1,2)\!: c=-m+2\)  ·  \((2,3)\!: c=-2m+3\)  ·  \((3,4)\!: c=-3m+4\)
Intersect the first two
\(-m+2 = -2m+3 \;\Rightarrow\; m = 1,\quad c = -1+2 = 1\)
Test the third line at (m, c) = (1, 1)
\(c = -3(1)+4 = 1\) ✓ — it passes through the same point.
Conclude
All three vote-lines meet at \((m,c)=(1,1)\), so the points are collinear, lying on \(y = x + 1\).
Answer

Collinear — the common line is \(y = x + 1\) (slope \(m=1\), intercept \(c=1\)).

Sanity-check it: in the demo above, click "Load (1,2)(2,3)(3,4)" — the three coloured vote-lines cross at a single green point, and \(y=x+1\) is drawn back in image space.

Key takeaways

◆ Remember these
  • Canny = Gaussian smooth → gradient mag/angle → non-max suppression → double-threshold hysteresis (ratio 2:1–3:1).
  • Hough is voting: each point votes for every line through it; peaks in parameter space are the detected lines.
  • Image point \((x_0,y_0)\) ⇒ parameter line \(c=-x_0 m + y_0\); collinear points ⇒ vote-lines share a common intersection.
  • Use polar \(\rho=x\cos\theta+y\sin\theta\) so vertical lines (infinite slope) still fit a bounded accumulator.
