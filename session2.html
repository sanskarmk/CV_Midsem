The simple image-formation model

We write an image as a 2D function \(f(x,y)\). Its value is proportional to the energy radiated from the scene at that point, and it factors into two physically meaningful parts:

\[ f(x,y) = i(x,y)\,\cdot\,r(x,y) \]
  • \(i(x,y)\) — illumination: the amount of source light falling on the scene, \(0 \le i(x,y) < \infty\).
  • \(r(x,y)\) — reflectance: the fraction of light the surface bounces back, \(0 \le r(x,y) \le 1\), where 0 is total absorption (black) and 1 is total reflectance (white).

For a monochrome (grayscale) image the intensity at \((x,y)\) is called the gray level \(\ell\), scaled to the range \([0, L-1]\). All values in between are shades of gray from black (0) to white (\(L-1\)). For an 8-bit image, \(L = 256\) so levels run \(0\ldots255\).

◆ Why the split matters

Separating illumination from reflectance is the basis of many corrections: if uneven lighting \(i(x,y)\) is the problem, you can estimate and divide it out to recover the true reflectance — which is the property of the object you actually care about.

Pinhole camera vs lens

The simplest camera is a pinhole: a single tiny hole projects an inverted image. It has perfect "focus" everywhere but lets in almost no light. So why do real cameras use lenses?

◆ The quiz answer

Because lenses gather far more light than a pinhole, dramatically reducing the exposure time needed for a bright, low-noise image. (The trade-off: lenses introduce focus/depth-of-field limits and optical distortions — topics for camera-calibration later.)

Sampling & quantization

A real scene is continuous in both position and brightness. To store it digitally we discretize both:

Sampling

Digitizing the coordinate values — deciding how many pixels (the spatial grid). Take a scan line through the scene and read it at evenly spaced positions.

Quantization

Digitizing the amplitude values — deciding how many gray levels each sample may take (the bit depth, \(L = 2^b\)).

The continuous image projected onto a sensor array is sampled (one value per sensor cell) and each value is then rounded to the nearest available level. The result is the matrix of integers we call a digital image.

▲ Exam-style definition

If asked to "explain sampling and quantization" (this is literally Midsem Q1a): Sampling = discretizing spatial coordinates (x, y); Quantization = discretizing the intensity/amplitude. Too little sampling → blocky pixelation; too little quantization → false contouring (visible banding in smooth regions).

Matrix representation

After sampling and quantization, an \(M \times N\) image is the matrix

\[ f = \begin{bmatrix} f(0,0) & \cdots & f(0,N-1)\\ \vdots & \ddots & \vdots \\ f(M-1,0) & \cdots & f(M-1,N-1) \end{bmatrix} \]

Storage (uncompressed) is simply \(M \times N \times b\) bits. This is a common sub-question: a \(512\times512\), 8-bit image needs \(512\cdot512\cdot8 = 2{,}097{,}152\) bits \(= 256\) KB.

Elementary intensity transformations

These are point operations: the output pixel depends only on the input pixel through a mapping \(s = T(r)\), where \(r\) is input intensity and \(s\) is output. The transformation curve is the whole story.

TransformFormulaUse
Negative\(s = (L-1) - r\)Reveal detail in bright regions (e.g. mammograms).
Log\(s = c\,\log(1+r)\)Compress huge dynamic range; expand dark values. Used to display Fourier spectra.
Power-law (gamma)\(s = c\,r^{\gamma}\)Gamma correction for displays; \(\gamma<1\) brightens, \(\gamma>1\) darkens.
Contrast stretchpiecewise-linearSpread a narrow intensity range over the full scale.
Thresholding\(s = (L-1)\) if \(r \ge T\) else 0Binarize; limiting case of contrast stretch.
Intensity-level slicinghighlight \([A,B]\)Emphasise a specific band of intensities.
◆ Gamma, intuitively

CRT displays respond to voltage by a power law with exponent ≈ 1.8–2.5, so an uncorrected image looks too dark. Gamma correction pre-applies the inverse power so the displayed image looks right. Same maths is used to brighten dark MRI detail (\(\gamma<1\)) or tame washed-out aerial photos (\(\gamma>1\)).

Worked numerical — contrast stretching

This is exactly Midsem Q1(b). A linear (full-range) contrast stretch maps an input range \([r_{\min}, r_{\max}]\) onto \([0, L-1]\):

\[ s = (r - r_{\min}) \cdot \frac{L-1}{\,r_{\max} - r_{\min}\,} \]

Problem. An image has pixel values in \([50, 150]\). Stretch to the full 8-bit range \([0, 255]\). What is the new value of a pixel originally at \(r = 100\)?

Set up the slope
\(\dfrac{L-1}{r_{\max}-r_{\min}} = \dfrac{255}{150-50} = \dfrac{255}{100} = 2.55\)
Apply to r = 100
\(s = (100 - 50)\times 2.55 = 50 \times 2.55 = 127.5\)
Round to an integer level
\(s \approx \mathbf{128}\) (or 127). The mid-range input lands at mid-range output — as expected for a linear stretch.
Answer

New intensity \(= 127.5 \approx 128\).

Key takeaways

◆ Remember these
  • \(f(x,y) = i(x,y)\,r(x,y)\): illumination × reflectance, with \(r \in [0,1]\).
  • Sampling = discretize position; quantization = discretize intensity (\(L=2^b\)).
  • Spatial resolution = detail (pixels); intensity resolution = brightness steps (levels).
  • Point transforms: negative \((L{-}1){-}r\), log \(c\log(1{+}r)\), gamma \(c\,r^{\gamma}\), linear stretch \((r-r_{\min})\frac{L-1}{r_{\max}-r_{\min}}\).
