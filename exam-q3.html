Edge-Detection Kernel & Color Models

📄 Original question

A simplified edge detection kernel is given as \(\begin{bmatrix}-1&0&1\\-1&0&1\\-1&0&1\end{bmatrix}\)

Q3(a) What type of edges does this kernel primarily detect? 1M

Q3(b) Apply the previous kernel to the central pixel of the following image patch \(\begin{bmatrix}12&15&18\\15&10&15\\18&15&12\end{bmatrix}\) 2M

Q3(c) Compare and contrast the RGB and CMYK color models. Discuss their applications and explain why different models are used for different purposes. 2M

Solution — Q3(a)

The kernel subtracts the left column from the right column — it is the Prewitt \(g_x\) operator, computing a horizontal intensity gradient \(\partial f/\partial x\). A horizontal gradient is large where intensity changes left-to-right, i.e. across a vertical edge.

Answer

It primarily detects vertical edges (it responds to horizontal intensity changes; it's a Prewitt horizontal-gradient \(g_x\) mask).

Solution — Q3(b)

Slide the kernel over the patch, multiply element-wise and sum (correlation). The kernel's centre sits on the central pixel (value 10):

Patch 121518 151015 181512 Kernel -101 -101 -101
Row 1
\((-1)(12)+(0)(15)+(1)(18) = -12+18 = 6\)
Row 2
\((-1)(15)+(0)(10)+(1)(15) = -15+15 = 0\)
Row 3
\((-1)(18)+(0)(15)+(1)(12) = -18+12 = -6\)
Sum
\(6 + 0 + (-6) = \mathbf{0}\)
Answer

The response is \(\mathbf{0}\). The patch is symmetric left-to-right (column 1 and column 3 are mirror images: 12/18, 15/15, 18/12), so there is no net horizontal gradient — no vertical edge here. (Convolution, which flips the kernel, also gives 0.)

Solution — Q3(c)

RGBCMYK
TypeAdditive (adds emitted light)Subtractive (inks absorb light)
PrimariesRed, Green, BlueCyan, Magenta, Yellow + blacK
All channels max →WhiteBlack
All channels min →BlackWhite (bare paper)
ApplicationMonitors, cameras, screensColor printing
Relation\(C=1-R,\; M=1-G,\; Y=1-B\);   \(K=\min(C,M,Y)\)
Model answer

RGB is additive — colors are made by emitting and adding red, green, blue light (used by displays and cameras); all three at maximum gives white, all off gives black. CMYK is subtractive — inks on paper absorb (subtract) light (used by printers); more ink gives darker, bare paper is white. They are complementary (\(C=1-R\), etc.), and \(K\) (black) is added to save colored ink and deepen blacks. Different models are used because the physical process differs: screens build color by emitting light (start black, add), while print builds color by absorbing light (start white, subtract).

P3.1 Apply the same kernel \([-1,0,1;-1,0,1;-1,0,1]\) to the central pixel of \(\begin{bmatrix}10&10&200\\10&10&200\\10&10&200\end{bmatrix}\). 2M
Each row: right − left
\((200-10)+(200-10)+(200-10)\)
Sum
\(190+190+190 = \mathbf{570}\)
Answer

Response \(= 570\) — a strong positive value, because there's a genuine vertical edge (dark → bright, left → right). Contrast with the symmetric exam patch which gave 0.

P3.2 What does the kernel \(\begin{bmatrix}-1&-1&-1\\0&0&0\\1&1&1\end{bmatrix}\) detect? Apply it to \(\begin{bmatrix}20&20&20\\20&20&20\\200&200&200\end{bmatrix}\). 2M

This is the Prewitt \(g_y\) (bottom row − top row) → detects horizontal edges.

Bottom row − top row, per column
\((200-20)\times 3\)
Sum
\(180\times 3 = \mathbf{540}\)
Answer

Detects horizontal edges; response \(=540\) on this top-dark / bottom-bright patch.

P3.5 Why is HSI/HSV often better than RGB for color-based image segmentation? 1M
Answer

HSI/HSV decouples color (Hue, Saturation) from brightness (Intensity). So you can select "all reddish pixels" via Hue alone, robust to lighting/shadow changes — whereas in RGB a change in brightness alters all three channels at once, making a single color hard to isolate.
